# 800 AI

Use this course to introduce AI only after the reader understands Stackpress
runtime, data, project structure, schema, generation, built-ins, and admin
surfaces.

AI should feel like an advanced automation layer, not the first reason to use
Stackpress.

## 800 AI

Stackpress AI can expose tools, generate artifacts, respond to events, and help
agents work through Stackpress development phases.

**Checkpoint**

The reader can explain where AI fits in the Stackpress workflow.

## 810 MCP

MCP-style transports expose Stackpress capabilities to agent tools.

Teach the purpose and the operational path. Keep protocol and transport details
in reference.

**Course work**

- run or inspect one MCP transport
- understand which Stackpress capabilities are exposed

## 811 stdio Transport

The stdio transport is useful for local agent integration.

**Course work**

- run or inspect the stdio script
- understand where local agent communication enters Stackpress

## 820 Artifacts

AI-generated artifacts should be treated like generated work: inspect them,
verify them, and decide what becomes source.

**Course work**

- generate or inspect one artifact
- decide whether it should become committed source

## 830 Hooks

Hooks and events let AI participate in runtime or generation workflows.

**Course work**

- trace one AI-related event or transform hook

## 831 AI Events

AI events are runtime extension points.

**Course work**

- register or inspect one AI event

## 832 Transform Hooks

Transform hooks are generation-time extension points.

**Course work**

- inspect a hook that writes generated output

## 840 Skills

Stackpress skills are portable agent workflows. They help an agent move from a
plain-English product idea to a working Stackpress app.

The skill set covers:

- discovery
- coordination
- scaffolding
- idea authoring
- plugin routing
- plugin implementation
- verification

## 841 Skill Workflow

The recommended workflow is:

1. `stackpress-app-discovery`
2. `stackpress-app-coordinator`
3. `stackpress-app-scaffold`
4. `stackpress-idea-authoring`
5. run generation
6. `stackpress-plugin-router`
7. plugin implementation skills as needed
8. `stackpress-app-verification`

**Checkpoint**

The reader can map a product request to the next skill in the workflow.

## 842 App Discovery Skill

`stackpress-app-discovery` turns a vague request into a buildable app brief.

**Course work**

- clarify audience
- identify entities
- identify flows
- identify auth/admin/data needs

## 843 App Coordinator Skill

`stackpress-app-coordinator` manages the end-to-end Stackpress build flow.

**Course work**

- sequence scaffold, schema, generation, plugin work, and verification

## 844 Scaffold Skill

`stackpress-app-scaffold` creates baseline app files from its embedded scaffold
asset.

**Course work**

- inspect what the scaffold creates
- understand which files are meant to be edited next

## 845 Idea Authoring Skill

`stackpress-idea-authoring` drafts and refines `schema.idea`.

**Course work**

- model a domain in idea syntax
- validate Stackpress idea patterns

## 846 Plugin Skills

Plugin skills help decide and implement the correct extension layer:

- `stackpress-plugin-router`
- `stackpress-plugin-scaffold`
- `stackpress-plugin-pages-events`
- `stackpress-plugin-views`
- `stackpress-plugin-idea-generator`

**Course work**

- decide whether a feature belongs in schema, runtime plugin code, page/view
  code, or generator code

## 847 Verification Skill

`stackpress-app-verification` checks a phase before the workflow moves on.

**Course work**

- verify scaffold files
- verify schema readiness
- verify generated output
- verify plugin wiring
- verify runtime behavior
