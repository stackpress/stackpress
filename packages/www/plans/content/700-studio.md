# 700 Studio

Use this course as future-facing content until Studio behavior is fully defined.
The repository already has generated admin and client surfaces, so the first
Studio content should focus on inspecting and shaping generated app behavior.

## 700 Studio

Studio should add a visual or assisted layer on top of Stackpress schema,
fields, relations, generated admin, and import/export workflows.

**Checkpoint**

The reader understands what Studio adds beyond code-first development.

## 710 Schema Explorer

A schema explorer should help readers inspect models, fields, relations, and
generated artifacts without opening every generated file manually.

**Course work**

- inspect models
- inspect generated artifacts
- connect visual schema state back to `schema.idea`

## 720 Fields

Field editing should make type, metadata, validation, and generated UI effects
visible.

**Course work**

- modify a field
- review generated effects

## 721 Field Validation

Validation connects authoring metadata to generated behavior.

**Course work**

- add or inspect a validator
- verify generated form/action behavior

## 730 Relations

Relations should be visible as app modeling choices, not only database details.

**Course work**

- create or inspect a relation
- understand its generated output

## 740 Generated Admin

Generated admin is the practical bridge between schema and management screens.

**Course work**

- generate admin output
- inspect generated pages and views

## 741 Admin Pages

Admin pages are generated route surfaces for model-oriented workflows.

**Course work**

- find an admin page route
- identify where customization belongs

## 742 Admin Views

Admin views are generated presentation surfaces.

**Course work**

- inspect one generated admin view
- trace it back to schema metadata

## 743 Admin Client

Admin client output helps inspect generated behavior in readable TypeScript.

**Course work**

- inspect admin client output
- use it for troubleshooting

## 750 Import / Export

Import/export supports moving structured app data in and out.

**Course work**

- inspect or run one import/export flow
