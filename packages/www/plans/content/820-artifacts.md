# 820 Artifacts

Treat AI-generated artifacts as generated work that must be inspected, verified, and either accepted into source or discarded. The same idea shows up through inspectable project surfaces.

**Previously:** The previous lesson, `811 stdio Transport`, gave you the setup this page builds on. Here, the focus shifts to `Artifacts` so you can place the next Stackpress surface in the course path.

## 820.1. What You Are Looking For

AI-generated artifacts can be helpful drafts, but they are not automatically source of truth. Each artifact needs a decision: keep it as source, inspect it as output, or discard it.

## 820.2. Where Artifacts Live

Use this review path:

 1. Identify what created the artifact.
 2. Decide whether it belongs in source or generated output.
 3. Compare it against specs, package code, and templates.
 4. Run the smallest verification command that proves it works.
 5. Keep or discard it intentionally.
