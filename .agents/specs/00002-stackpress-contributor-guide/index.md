# Stackpress Contributor Guide Research

This is the entry point for the research spec that will derive a comprehensive,
source-backed contribution guide for the Stackpress monorepo.

## Load Map

- [Brief](brief.md): load for the user goal, scope, non-goals, evidence boundary,
  and intended guide capability.
- [Status](status.md): load for phase progress, Freeze blockers, clarification
  state, and the next action.
- [Decisions And Gaps](decisions.md): load for accepted planning decisions,
  research questions, assumptions to challenge, and user-owned clarifications.
- [Setup Decisions](setup-decisions.md): load for the five accepted decisions
  that established the research method before source investigation.
- [Research](research.md): load before inspecting package source, comparing
  patterns, or recording findings.
- [Phase 1 Package Inventory](phase1-package-inventory.md): load for the current
  workspace set, build/aggregate distinctions, source-shape inventory, and
  classification conflicts discovered before pattern interpretation.
- [Phase 2 Pattern Matrix](phase2-pattern-matrix.md): load for package anatomy
  classes, exact source-placement evidence, stable conventions, variations, and
  inconsistencies that must not become rules.
- [Phase 3 Contract Traces](phase3-contract-traces.md): load for execution phases,
  generator discovery, generated/runtime pairing, page/view routing, aggregate
  composition, and verification chains.
- [Maintainer Clarifications](phase4-clarifications.md): load when answering the
  source-derived intent and policy questions that block final synthesis.
- [Phase 5 Contributor Model](phase5-contributor-model.md): load for the proposed
  contributor mental model, package-selection rules, execution boundaries, and
  current-versus-new-work authority.
- [Phase 5 Placement Guide](phase5-placement-guide.md): load for exact source
  destinations, companion changes, generator rules, and aggregate criteria.
- [Phase 5 Verification Guide](phase5-verification-guide.md): load for the
  proposed tests, coverage target, contract checks, and closeout evidence.
- [Phase 6 Retrieval Plan](phase6-retrieval-plan.md): load before cold-context
  agent testing or when recording retrieval failures and repairs.
- [Phase 6 Baseline Results](phase6-baseline-results.md): load for the cold-agent
  scores, failure classifications, and required guide repairs.
- [Phase 7 Promotion Proposal](phase7-promotion-proposal.md): load for the
  review-first KB file map, claim ownership, exclusions, and rerun gate.
- [Phase 8 Final Results](phase8-final-results.md): load for the accepted
  promotion, rerun sequence, final 9/9 score, and Freeze evidence.
- [Phase 8 Final Answers](phase8-final-answers.md): load for normalized accepted
  answers distilled from the final cold-agent responses.
- [Baseline Answers A](phase6-baseline-answers-a.md),
  [B](phase6-baseline-answers-b.md), and
  [C](phase6-baseline-answers-c.md): load only when verbatim retrieval evidence
  is needed.
- [Proofs](proofs.md): load when source evidence cannot settle a technical
  contract or when validating the guide against fresh agent sessions.

## Authority Boundary

The current KB and the frozen KB-population spec are research inputs, not
automatic answers for this spec. Current local source, tests, manifests,
generated artifacts, and observed behavior own checkout-specific findings.
Only accepted reusable conclusions may later be promoted into context or linked
detailed references.
