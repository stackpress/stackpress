# Setup Decisions

## DEC-001: Use A Separate Research Spec

Research the contributor model here without reopening the frozen KB-population
spec. Existing KB answers proved insufficient in separate agent sessions.

Status: Accepted by user instruction.

## DEC-002: Research The Complete Contribution System

Treat the four reported topics as initial failure evidence, not the scope
boundary. Include exact source locations, implementation considerations, and
patterns shared by packages.

Status: Accepted by user instruction.

## DEC-003: Derive Rules Through Comparison

Compare packages and contract chains before labeling a pattern required or
recommended. Repetition can be a framework contract, convention, specialization,
copied history, or inconsistency.

Status: Accepted for the research method.

## DEC-004: Clarify Before Uncertain Promotion

Ask focused questions when source and tests do not establish maintainer intent,
policy, or the preferred contributor rule. Do not fill gaps with plausible
assumptions.

Status: Accepted by user instruction.

## DEC-005: Validate Retrieval In Fresh Sessions

Test the eventual guide using realistic contribution questions in fresh agent
sessions before KB promotion. Generic or path-guessing answers are failures.

Status: Accepted for this spec.
