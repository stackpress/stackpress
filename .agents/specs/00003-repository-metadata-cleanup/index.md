# Repository Metadata Cleanup

This spec owns the narrow maintenance pass recommended after the contributor KB
research.

## Load Map

- [Brief](brief.md): user goal, scope, non-goals, and affected repository surfaces.
- [Status](status.md): phase state, blockers, and next action.
- [Decisions And Gaps](decisions.md): accepted cleanup decisions and the runtime
  CLI verification question.
- [Research](research.md): source/history findings behind each proposed repair.
- [Proofs](proofs.md): why prototypes are not currently required.
- [Implementation Sprint](tasks/sprint.md): load for the proposed two-task edit
  sequence, verification, and acceptance criteria.

## Authority

Current source, tracked files, manifests, workspace resolution, local Git history,
and packaging/build behavior own this cleanup. The contributor KB supplies the
accepted package classifications and exception boundaries.
