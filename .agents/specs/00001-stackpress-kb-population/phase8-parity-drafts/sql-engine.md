# SQL Engine

`Engine` is the execution boundary between Inquire's structured SQL builders and
a Stackpress database connection. It retains the selected connection and
dialect, creates six builder families, executes parameterized or tagged SQL,
and coordinates transaction work.

```ts
import { Engine } from 'stackpress/sql'

const users = await new Engine(connection)
  .select('*')
  .from('user')
  .where('active = ?', [true])
  .orderBy('name', 'asc')
  .limit(20)
```

The engine creates `Create`, `Alter`, `Insert`, `Select`, `Update`, and `Delete`
builders. Builders are thenable: awaiting one builds the dialect-specific SQL
and executes it through the engine. They can also expose their built query for
inspection without execution.

## Direct Execution

Use `query` for an explicit query/value pair and `sql` for tagged templates.
Values remain separate from SQL text so the connection can format placeholders
for its native driver. The engine's `before` hook can observe or short-circuit
execution, which makes it suitable for logging, policy, caching, or test seams.

```ts
const result = await engine.query(
  'SELECT * FROM user WHERE email = ?',
  ['person@example.com']
)
```

Transactions run against a connection-owned transaction resource. Multi-query
DDL operations generated by alter/diff workflows should remain in that same
transaction context.

## Schema Operations And Limits

The engine supplies drop, rename, truncate, and schema-diff operations in
addition to CRUD builders. A diff reports structural change; it does not infer
semantic renames, and table names are not themselves part of the compared field
shape. Raw clauses remain caller-owned SQL and require the same care as direct
queries.

The dialect determines rendered syntax while the connection determines native
execution and result shape. Equivalent behavior across MySQL, PostgreSQL, and
SQLite does not imply textually identical SQL.
