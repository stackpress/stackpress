# Phase 6 Decisions And Gaps

## Accepted Planning Decisions

### P6-DEC-001: Use retrieval questions as taxonomy owners

Decision: Propose eight context files for identity, architecture, modeling,
runtime, interfaces, ecosystem, contribution, and compatibility.

Reason: These domains route durable questions across all six artifact families
without mirroring packages or duplicating truth by output type.

Status: Accepted by the user's instruction to proceed; promoted and validated.

### P6-DEC-002: Keep marketing as an output

Decision: Do not create a marketing Context File. Marketing should combine
accepted identity, architecture, ecosystem, and compatibility facts with
founder-approved positioning.

Reason: The code supports message evidence but cannot choose category, promise
priority, or branded terminology.

Status: Accepted by the user's instruction to proceed.

### P6-DEC-003: Promote technical vocabulary with purpose labels

Decision: Technical phrases may enter context as architecture explanations while
remaining explicitly unapproved as public taglines.

Reason: This allows future agents to explain the system consistently without
turning research language into inferred brand strategy.

Status: Accepted by the user's instruction to proceed.

### P6-DEC-004: Keep absent governance out of capability claims

Decision: Context may record that guarantees were not found and explain the
risk, but proposed registries, handshakes, matrices, and review gates must remain
spec-local.

Reason: They are recommendations, not current Stackpress behavior.

Status: Accepted by existing promotion rules; apply during drafting.

## Gap Dispositions

### GAP-003: Artifact robustness

Finding: The proposed taxonomy routes documentation, article, diagram, teaching,
marketing, and contributor artifacts. Marketing remains conditional on founder
positioning decisions.

Status: Resolved by `phase6-context-validation.md`.

### GAP-004: Project language

Finding: Technical vocabulary is classified, but public category, lead promise,
and branded terminology remain founder decisions.

Status: Technical vocabulary accepted; founder/public language explicitly
deferred by user acceptance at Freeze.

### GAP-005: Replacement taxonomy

Finding: `phase6-taxonomy.md` proposes a retrieval-tested eight-file replacement.

Status: Resolved. The user instructed the approved next promotion step, and the
eight-file taxonomy was promoted without adjustment.

## Promotion Status

All eight Context Files and the read-first router are promoted. Technical
artifact retrieval passes from context alone. Founder positioning remains open
and is explicitly separated from technical vocabulary; future agents need not
open this spec for accepted technical Stackpress truth.
