# Intersection Points

<!-- agent-workspace-rules:start -->
Load this Reference File when running an Intersection Scan during Source Material ingestion.

Start with these intersection points:

1. `.agents/context/index.md`: context map entries, missing destination links, renamed context topics, and read-first routing.
2. `.agents/context/**/*.md`: Accepted Reusable Truth, product facts, constraints, terminology, decisions, gaps, open questions, and contradictions.
3. `.agents/references/**/*.md`: supporting detail, Source Provenance, prior Source Material, split evidence, long examples, and rationale related to the import.
4. `.agents/specs/**/*.md`: requirements, acceptance criteria, decisions, risks, proposed tasks, validation records, and unresolved assumptions affected by the import.
5. `.agents/development/**/*.md`: active progress, blockers, handoffs, QA notes, implementation claims, and pending work affected by the import.
6. `.agents/AGENTS.md` and `.agents/TERMS.md`: workspace rules and terminology. Update these only when the import is about the Agent Workspace itself.
7. `.agents/workflows/**/*.md`: reusable process rules. Update these only when the import is about Agent Workspace workflows.
8. Other `.agents/**/*.md`: project-specific Agent Files. Classify by folder, owner, and purpose before treating matches as Accepted Reusable Truth.
